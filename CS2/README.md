# Getting Started with Your Code Repository


## Purpose of csgit.stolaf.edu
This is a hosted version control system and code repository that is for your use during this course.  

You will use it to submit programs that you write for many assignments (Homework and Labs especially) in this course.  It is essential that you only submit your own code to this repository to maintain Academic Honesty.  

We also use this system for several other courses at St. Olaf College such as software development, algorithms, and machine learning courses.  If you work in software development after graduation, you will also use a system similar to this one daily.  It is worth the effort to master this submission system. 

## Instructions for Using Your Code Repository
Instructions for working with this repository are included in the in-class exercises during the first three weeks of the course and Homework 7.  

## Troubleshooting  
There are two common problems that surface in using csgit.stolaf.edu:
1. You run out of disk space on the lab computer network.  There is a lot of computer space used in maintaining the history of interactions between csgit and the computer network in the labs.  If you have run out of space, the two systems often cannot "talk" to one another, and you will receive error messages that may be quite cryptic.  Check your Disk Quota before anything other attemps at troubleshooting.
2. You may have merge conflicts, especially if you use multiple computers for writing code.  This is a common occurance in using any version control system.  You will need to help git understand which version of a file you want to upload.  Ask your instructor for help or search for "resolving merge conflicts with git".
